## Overview

Lichess Studies provide powerful tools for analyzing games, creating chess lessons, and collaborating with others in real-time. Studies support multiple chapters, rich annotations, variations, and can be shared publicly or kept private.

## Study Data Model

```scala
// modules/study/src/main/Study.scala:11
case class Study(
    id: StudyId,
    name: StudyName,
    members: StudyMembers,
    position: Position.Ref,      // Current chapter and path
    ownerId: UserId,
    visibility: Visibility,      // public, unlisted, or private
    settings: Settings,
    from: Study.From,            // Source (scratch, game, relay)
    likes: Study.Likes,
    description: Option[String] = None,
    topics: Option[StudyTopics] = None,
    flair: Option[Flair] = None,
    createdAt: Instant,
    updatedAt: Instant
):
  val slug = scalalib.StringOps.slug(name.value)
  
  def owner = members.get(ownerId)
  
  def isOwner[U: UserIdOf](u: U) = ownerId.is(u)
  
  def canContribute[U: UserIdOf](u: U): Boolean =
    isOwner(u) || members.get(u.id).exists(_.canContribute) || 
      u.is(UserId.lichess)
  
  def canView(id: Option[UserId]) = !isPrivate || id.exists(members.contains)
```

```scala
// modules/study/src/main/Study.scala:159
def make(
    user: User,
    from: From,
    id: Option[StudyId] = None,
    name: Option[StudyName] = None,
    settings: Option[Settings] = None
) =
  val owner = StudyMember(id = user.id, role = StudyMember.Role.Write)
  Study(
    id = id | makeId,
    name = name | StudyName(s"${user.username}'s Study"),
    members = StudyMembers(Map(user.id -> owner)),
    position = Position.Ref(StudyChapterId(""), UciPath.root),
    ownerId = user.id,
    visibility = Visibility.unlisted,
    settings = settings | Settings.init,
    from = from,
    likes = Likes(1),
    createdAt = nowInstant,
    updatedAt = nowInstant
  )
```

## Computer Analysis in Studies

Request Stockfish analysis for study chapters:

```scala
// modules/fishnet/src/main/Analyser.scala:86
def study(req: lila.core.fishnet.Bus.StudyChapterRequest): Fu[Analyser.Result] =
  analysisRepo.chapterExists(req.chapterId)
    .flatMap:
      if _ then fuccess(Analyser.Result.NoChapter)
      else
        val sender = Sender(req.userId, none, mod = false, system = false)
        val limitFu =
          if req.official then fuccess(Analyser.Result.Ok)
          else limiter(sender, ignoreConcurrentCheck = true, ownGame = false)
        limitFu.flatMap: result =>
          val work = makeWork(
            game = Work.Game(
              id = chapterId.value,
              initialFen = initialFen,
              studyId = studyId.some,
              variant = variant,
              moves = moves.take(maxPlies).map(_.uci).mkString(" ")
            ),
            startPly = Ply(initialFen.map(_.colorOrWhite).so(_.fold(0, 1))),
            sender = sender,
            origin = if req.official then Origin.officialBroadcast 
                     else Origin.manualRequest
          )
```

<Warning>
  Study chapter analysis is subject to the same rate limits as game analysis, except for official broadcasts.
</Warning>

## Study Likes & Ranking

```scala
// modules/study/src/main/Study.scala:110
opaque type Likes = Int

case class Liking(likes: Likes, me: Boolean)

opaque type Rank = Instant
object Rank extends OpaqueInstant[Rank]:
  def compute(likes: Likes, createdAt: Instant) =
    Rank(createdAt.plusHours(likesToHours(likes)))
  private def likesToHours(likes: Likes): Int =
    if likes < 1 then 0
    else (5 * math.log(likes) + 1).toInt.min(likes) * 24

def notable = likes.value > 10 || {
  likes.value > 5 && createdAt.isBefore(nowInstant.minusDays(7))
}
```

Studies with more likes:
- Rank higher in search
- Appear in "Popular Studies"
- Get featured more prominently

## Cloning Studies

```scala
// modules/study/src/main/Study.scala:68
def cloneFor(user: User): Study =
  val owner = StudyMember(id = user.id, role = StudyMember.Role.Write)
  copy(
    id = Study.makeId,
    members = StudyMembers(Map(user.id -> owner)),
    ownerId = owner.id,
    visibility = Visibility.`private`,
    from = Study.From.Study(id),
    likes = Likes(1),
    createdAt = nowInstant,
    updatedAt = nowInstant
  )
```

Cloning allows:
- Personal copy of public studies
- Starting point for your own analysis
- Preserving content that might be deleted

<Note>
  Cloned studies are always created as private. You can change visibility after cloning.
</Note>
