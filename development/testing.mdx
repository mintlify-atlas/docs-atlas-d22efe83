## Testing Frameworks

Lila uses multiple testing frameworks depending on the module:

### MUnit

The primary testing framework for most modules:

```scala
// Dependencies.scala
val munit = "org.scalameta" %% "munit" % "1.2.4" % Test
val munitCheck = "org.scalameta" %% "munit-scalacheck" % "1.2.0" % Test
```

### ScalaCheck

Property-based testing for complex scenarios:

```scala
val scalacheck = "org.scalacheck" %% "scalacheck" % "1.19.0" % Test
```

### Chess Test Kit

Specialized testing for chess logic:

```scala
val testKit = "com.github.lichess-org.scalachess" %% "scalachess-test-kit" % version % Test
```

## Running Tests

### All Tests

```bash
./lila.sh
```

In the sbt console:

```scala
test              // Run all tests
testOnly          // Run specific test
testQuick         // Run only failed tests
```

### Module-Specific Tests

```scala
// Test a specific module
game/test
study/test
tournament/test

// Test a specific file
testOnly lila.game.PgnDumpTest
testOnly lila.tournament.PairingTest
```

### Continuous Testing

```scala
~test             // Re-run tests on file changes
~testQuick        // Re-run failed tests on changes
~testOnly lila.game.*  // Watch specific tests
```

## Test Structure

### MUnit Test Example

```scala
package lila.game

import chess.*
import chess.format.pgn.{ SanStr }

class PgnDumpTest extends munit.FunSuite:

  test("roundtrip pgns"):
    val pgn = "e4 e5 Nf3 Nc6 Bb5"
    val sanStrs = pgn.split(' ').toList.map(SanStr(_))
    val output = PgnDump.makeTree(sanStrs, Vector.empty, Color.White)
    assertEquals(output.isDefined, true)

  test("pgn from few moves"):
    val moves = "e5 Ke2 Ke7".split(' ').toList.map(SanStr(_))
    val tree = PgnDump.makeTree(moves, Vector.empty, Color.White)
    assert(tree.isDefined)
```

### Test Fixtures

Many modules include test fixtures for common scenarios:

```scala
// modules/study/src/test/PgnFixtures.scala
object PgnFixtures:
  val complex = """1. e4 e5 2. Nf3 Nc6 3. Bb5 a6 ..."""
  val tactics = """1. e4 e5 2. Nf3 f6 3. Nxe5 fxe5 ..."""
```

### ScalaCheck Property Tests

```scala
import org.scalacheck.Prop.*

class BinaryTest extends munit.ScalaCheckSuite:
  
  property("encode and decode piece"):
    forAll { (piece: Piece) =>
      BinaryPiece.decode(BinaryPiece.encode(piece)) == piece
    }
```

## Common Test Patterns

### Testing with Dependencies

Modules can depend on test code from other modules:

```scala
// build.sbt
lazy val rating = module("rating",
  Seq(db, ui),
  tests.bundle
).dependsOn(common % "test->test")  // Use common's test code
```

### Integration Tests

For tests requiring multiple modules:

```scala
class StudyIntegrationTest extends munit.FunSuite:
  
  val db = TestDatabase.setup()
  val userRepo = UserRepo(db)
  val studyRepo = StudyRepo(db)
  
  test("create and retrieve study"):
    // Test cross-module functionality
```

### Async Tests

Lila is fully asynchronous - tests handle Futures:

```scala
import scala.concurrent.Future

test("async operation"):
  val result: Future[String] = someAsyncOperation()
  result.map { value =>
    assertEquals(value, "expected")
  }
```

## Manual Testing

### Local Development Server

1. Start the application:
   ```bash
   ./lila.sh
   run
   ```

2. Open browser: `http://localhost:9663`

3. Test features:
   - Play games
   - Create studies
   - Run tournaments
   - Test analysis board

### Testing with MongoDB

Some tests require MongoDB:

```bash
# Start MongoDB
mongod --dbpath /path/to/data

# Run tests that use database
sbt "testOnly *DatabaseTest"
```

### Testing WebSocket Features

Requires [lila-ws](https://github.com/lichess-org/lila-ws) running:

```bash
# In lila-ws directory
sbt run

# Configure socket domain in conf/application.conf
net.socket.domains = ["localhost:9664"]
```

## Performance Testing

### Benchmarking

For performance-critical code:

```scala
test("benchmark PGN parsing".ignore):  // Use .ignore for benchmarks
  val start = System.currentTimeMillis()
  (1 to 1000).foreach { _ =>
    PgnDump.parse(longPgn)
  }
  val elapsed = System.currentTimeMillis() - start
  println(s"Parsed 1000 games in ${elapsed}ms")
```

### Load Testing

For API endpoints and game logic:

- Use production-like data volumes
- Test with concurrent operations
- Monitor memory usage
- Check for resource leaks

## Continuous Integration

GitHub Actions runs tests on all pull requests:

```yaml
# .github/workflows/test.yml
name: Test
on: [push, pull_request]
jobs:
  test:
    runs-on: ubuntu-latest
    steps:
      - uses: actions/checkout@v2
      - name: Run tests
        run: sbt test
```

## Debugging Tests

### Print Debugging

```scala
test("debug test"):
  val result = someOperation()
  println(s"Result: $result")  // Print values
  assertEquals(result, expected)
```

### Test-Only Logging

Use test-specific logger configuration:

```xml
<!-- conf/logger.dev.xml -->
<logger name="lila.game" level="DEBUG"/>
```

### Selective Test Execution

```scala
test("skip this".ignore):  // Skip specific tests
  // Test code

test("only run this".only):  // Run only this test
  // Test code
```

## Best Practices

<AccordionGroup>
  <Accordion title="Keep tests fast">
    - Mock external services
    - Use in-memory databases when possible
    - Avoid Thread.sleep() in tests
  </Accordion>

  <Accordion title="Test one thing at a time">
    - Each test should verify a single behavior
    - Use descriptive test names
    - Keep assertions focused
  </Accordion>

    - ScalaCheck for complex logic
    - Generate edge cases automatically
    - Verify invariants hold

    - Invalid input
    - Boundary conditions
    - Race conditions
  </Accordion>

## Common Test Patterns

### Testing JSON Serialization

```scala
test("JSON roundtrip"):
  val game = makeGame()
  val json = gameWriter.writes(game)
  val parsed = json.validate[Game]
  assertEquals(parsed, JsSuccess(game))
```

### Testing Event Streams

```scala
test("game events"):
  val events = gameEventStream(gameId)
  val result = events
    .take(3)
    .runWith(Sink.seq)
  result.map { events =>
    assertEquals(events.length, 3)
  }
```

### Testing Permissions

```scala
test("user permissions"):
  assert(user.can(Permission.ModNote))
  assert(!user.can(Permission.Admin))
```
