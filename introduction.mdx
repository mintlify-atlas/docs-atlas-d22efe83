# Welcome to Lichess

Lichess (li[chess in sca]la) is a free, open-source online chess game server focused on [realtime](https://lichess.org/games) gameplay and ease of use. With over 4.7 billion games stored and analyzed, Lichess is one of the world's largest chess platforms.

<Note>
  Lichess is completely free and ad-free, funded by user donations. All rated games are published in a free PGN database.
</Note>

## What is Lichess?

Lichess provides a comprehensive chess platform with features ranging from casual play to professional analysis tools. The platform serves millions of players worldwide and offers:

- **Real-time gameplay** with WebSocket-based connections
- **Computer analysis** powered by Stockfish distributed across donated servers
- **Educational tools** including puzzles, lessons, and coordinate training
- **Community features** like tournaments, teams, forums, and broadcasts
- **Complete transparency** - all code is open source under AGPL-3.0

## Technology Stack

Lichess is built with modern, scalable technologies:

### Backend

- **Language**: [Scala 3](https://www.scala-lang.org/) - a modern, type-safe functional programming language
- **Framework**: [Play 2.8](https://www.playframework.com/) - reactive web framework
- **Database**: [MongoDB](https://www.mongodb.com) - storing over 4.7 billion games
- **Search**: [Elasticsearch](https://github.com/elastic/elasticsearch) - indexing games and content
- **Caching**: Redis, Caffeine, and Scaffeine for distributed and in-memory caching
- **Async Processing**: Akka streams and Scala Futures for fully asynchronous operation

```scala
// Example: The main entry point (app/Lila.scala:18)
object Lila:
  def main(args: Array[String]): Unit =
    lila.web.PlayServer.start(args): env =>
      LilaComponents(
        env,
        DefaultApplicationLifecycle(),
        Configuration.load(env)
      ).application
```

### Frontend

- **Language**: [TypeScript](https://www.typescriptlang.org/) for type-safe client code
- **UI Framework**: [snabbdom](https://github.com/snabbdom/snabbdom) - lightweight virtual DOM
- **Styling**: [Sass](https://sass-lang.com/) for CSS generation
- **Chess UI**: [@lichess-org/chessground](https://github.com/lichess-org/chessground) - custom chessboard component

```json
// Core frontend dependencies (package.json)
{
  "dependencies": {
    "@lichess-org/chessground": "^10.0.2",

    "chessops": "^0.15",

  }
}
```

### Chess Logic

- **Core Engine**: [scalachess](https://github.com/lichess-org/scalachess) - pure chess logic in Scala
- **Analysis**: [Stockfish](https://stockfishchess.org/) via fishnet cluster
- **Templating**: [scalatags](https://com-lihaoyi.github.io/scalatags/) for type-safe HTML

## Modular Architecture

Lichess uses a modular monolith architecture with 80+ modules (build.sbt:60-86):

```scala
// Module dependency hierarchy
lazy val modules = Seq(
  // Level 1 - Foundation
  core, coreI18n,
  // Level 2 - UI & Common utilities  
  ui, common, tree,
  // Level 3 - Infrastructure
  db, room, search,
  // Level 4 - Core services
  memo, rating,
  // Level 5 - Domain modules
  game, gathering, study, user, puzzle, analyse,
  report, pref, chat, playban, lobby, mailer, oauth,
  // Level 6+ - Feature modules
  tournament, round, swiss, relay, team, forum,
  // ... and many more
)
```

Each module is independently testable with clear dependency boundaries. The modular structure is reflected in the `modules/` directory with 85 separate subprojects.
