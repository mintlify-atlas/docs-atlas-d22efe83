## Overview

Lichess provides free, unlimited computer analysis for all games using Stockfish, one of the strongest chess engines in the world. The analysis is powered by Fishnet, a distributed computing network where volunteers contribute processing power.

## Analysis Architecture

### Analysis Request Flow

<Steps>

<Step title="User Requests Analysis">
  Player clicks "Request computer analysis" on a completed game
</Step>

<Step title="Request Validation">
  System checks rate limits and game eligibility
</Step>

<Step title="Work Queue">
  Analysis job added to Fishnet work queue
</Step>

<Step title="Client Acquisition">
  Available Fishnet client claims the work
</Step>

<Step title="Stockfish Computation">
  Client runs Stockfish on each position
</Step>

<Step title="Result Submission">
  Analysis results sent back and stored
</Step>

<Step title="Tree Building">
  Analysis integrated with game tree for display
</Step>

</Steps>

## Fishnet Distributed Network

### Analysis Request Handling

```scala
// modules/fishnet/src/main/Analyser.scala:34
def apply(
    game: Game,
    sender: Sender,
    originOpt: Option[Origin] = none
): Fu[Analyser.Result] =
  game.metadata.analysed
    .so(analysisRepo.exists(game.id))
    .flatMap:
      if _ then fuccess(Analyser.Result.AlreadyAnalysed)
      else if !gameApi.analysable(game) then fuccess(Analyser.Result.NotAnalysable)
      else
        val origin = originOpt.getOrElse:
          if sender.system then Origin.autoHunter else Origin.manualRequest
        limiter(sender, ownGame = game.userIds contains sender.userId)
          .flatMap: result =>
            (result.ok && dedup(game.id.value))
              .so:
                makeWork(game, sender, origin).flatMap: work =>
                  repo.getSimilarAnalysis(work)
                    .flatMap:
                      case Some(similar) if similar.isAcquired => funit
                      case _ =>
                        evalCache.skipPositions(work.game)
                          .flatMap: skipPositions =>
                            repo.addAnalysis(work.copy(skipPositions = skipPositions))
```

### Request Origins

Analysis can be triggered from multiple sources:

- **Manual Request**: Player clicks analysis button
- **Auto Hunter**: Anti-cheat system requests analysis
- **Auto Tutor**: Lichess Tutor automated insights
- **Official Broadcast**: Live tournament broadcast coverage

<Note>
  System-generated analysis requests (auto hunter, tutor) bypass concurrency limits and are prioritized differently in the queue.
</Note>

## Evaluation Cache

To optimize performance, Lichess maintains a distributed evaluation cache:

```scala
// modules/fishnet/src/main/Analyser.scala:69
evalCache
  .skipPositions(work.game)
  .monSuccess(_.fishnet.analysis.skipPositionsGame)
  .flatMap: skipPositions =>
    lila.mon.fishnet.analysis.evalCacheHits.record(skipPositions.size)
    repo.addAnalysis(work.copy(skipPositions = skipPositions))
```

### Cache Benefits

- **Faster analysis**: Cached positions skip Stockfish computation
- **Reduced load**: Volunteers' CPUs can focus on novel positions
- **Opening theory**: Common opening positions are pre-computed
- **Transposition handling**: Same position via different move orders

<Tip>
  The evaluation cache is particularly effective for opening positions and common endgames, often reducing analysis time by 30-50%.
</Tip>

## Analysis Data Structure

The `Analyser` component in the `analyse` module manages analysis storage:

```scala
// modules/analyse/src/main/Analyser.scala:9
final class Analyser(
    gameRepo: lila.core.game.GameRepo,
    analysisRepo: AnalysisRepo
)(using Executor)
    extends lila.tree.Analyser:

  def get(game: Game): Fu[Option[Analysis]] =
    analysisRepo.byGame(game)

  def save(analysis: Analysis): Funit =
    analysis.id match
      case Analysis.Id.Game(id) =>
        gameRepo.game(id).flatMapz: prev =>
          val game = prev.focus(_.metadata.analysed).replace(true)
          for
            _ <- gameRepo.setAnalysed(game.id, true)
            _ <- analysisRepo.save(analysis)
            _ <- sendAnalysisProgress(analysis, complete = true)
          yield Bus.pub(actorApi.AnalysisReady(game, analysis))
```

### Analysis Components

Each analysis contains:

- **Evaluations**: Centipawn evaluation for each position
- **Best moves**: Top engine moves for each position
- **Mate sequences**: Forced mate lines when detected
- **Classification**: Move quality (brilliant, good, inaccuracy, mistake, blunder)
- **Accuracy percentage**: Overall play quality metric

## Real-Time Progress Updates

```scala
// modules/analyse/src/main/Analyser.scala:35
def progress(analysis: Analysis): Funit = 
  sendAnalysisProgress(analysis, complete = false)

private def sendAnalysisProgress(analysis: Analysis, complete: Boolean): Funit =
  analysis.id match
    case Analysis.Id.Game(id) =>
      gameRepo.gameWithInitialFen(id).mapz: g =>
        Bus.pub(
          lila.tree.AnalysisProgress(
            id,
            () => makeProgressPayload(analysis, g.game, g.fen)
          )
        )
```

Progress updates are pushed via WebSocket as the Fishnet client completes each position.

## Study & Broadcast Analysis

Analysis works differently for study chapters:

```scala
// modules/fishnet/src/main/Analyser.scala:86
def study(req: lila.core.fishnet.Bus.StudyChapterRequest): Fu[Analyser.Result] =
  analysisRepo.chapterExists(req.chapterId)
    .flatMap:
      if _ then fuccess(Analyser.Result.NoChapter)
      else
        val sender = Sender(req.userId, none, mod = false, system = false)
        val limitFu =
          if req.official then fuccess(Analyser.Result.Ok)
          else limiter(sender, ignoreConcurrentCheck = true, ownGame = false)
        limitFu.flatMap: result =>
          val work = makeWork(
            game = Work.Game(
              id = chapterId.value,
              initialFen = initialFen,
              studyId = studyId.some,
              variant = variant,
              moves = moves.take(maxPlies).map(_.uci).mkString(" ")
            ),
            startPly = Ply(initialFen.map(_.colorOrWhite).so(_.fold(0, 1))),
            sender = sender,
            origin = if req.official then Origin.officialBroadcast else Origin.manualRequest
          )
```

<Info>
  Official broadcast chapters (live tournament coverage) bypass rate limits and are prioritized in the analysis queue.
</Info>

### Eligibility Requirements

```scala
// modules/game/src/main/GameExt.scala:50
def analysable(g: Game) =
  g.replayable && g.playedPlies > 4 &&
    Game.analysableVariants(g.variant) &&
    !Game.isOldHorde(g)
```

Games must:
- Be completed (finished or aborted after moves)
- Have more than 4 plies (2 full moves)
- Use a supported variant
- Not be legacy Horde games (pre-2015 rule change)
