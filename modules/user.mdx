The **user** module manages user accounts, authentication, profiles, performance ratings, and user-related data. It's a core dependency for nearly every other module.

### Username Rules

```scala
object nameRules:
  // what new usernames should be like
  val newUsernameRegex = "(?i)[a-z][a-z0-9_-]{0,28}[a-z0-9]".r
  val newUsernamePrefix = "(?i)^[a-z].*".r
  val newUsernameSuffix = "(?i).*[a-z0-9]$".r
  val newUsernameChars = "(?i)^[a-z0-9_-]*$".r
  val newUsernameLetters = "(?i)^([a-z0-9][_-]?)+$".r
```

Username constraints:
- 2-30 characters
- Must start with a letter
- Must end with a letter or number
- Can contain letters, numbers, underscores, and hyphens
- No consecutive special characters

### Game Players Cache

High-performance caching for game participants:

```scala
// Hit by game rounds - heavily cached
object gamePlayers:
  private type PlayersKey = (PairOf[Option[UserId]], PerfKey)

  def apply(userIds: ByColor[Option[UserId]], perf: PerfKey): Fu[GameUsers] =
    cache.get(userIds.toPair -> perf)

  def loggedIn(userIds: ByColor[UserId], perf: PerfKey): Fu[GameUsers.LoggedIn] =
    apply(userIds.map(_.some), perf).dmap(_.requireLogged)
```

## Caching Strategy

The user module uses aggressive caching:

```scala
// From modules/user/src/main/Cached.scala
final class Cached(
    userRepo: UserRepo,
    cacheApi: CacheApi
)(using Executor):
  
  val rankingsOf = cacheApi.sync[PerfKey, Map[UserId, Int]](
    name = "user.rankingsOf",
    compute = perfKey => computeRankings(perfKey),
    default = _ => Map.empty,
    expireAfter = _.ExpireAfterWrite(15 minutes)
  )
```
