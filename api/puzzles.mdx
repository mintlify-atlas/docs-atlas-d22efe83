## Get Daily Puzzle

Get the daily puzzle.

```bash
GET /api/puzzle/daily
```

<ResponseExample>
```json
{
  "game": {
    "id": "abcd1234",
    "perf": {
      "key": "blitz",
      "name": "Blitz"
    },
    "rated": true,
    "players": [
      {
        "userId": "player1",
        "name": "Player1",
        "color": "white"
      },
      {
        "userId": "player2",
        "name": "Player2",
        "color": "black"
      }
    ],
    "pgn": "e4 e5 Nf3..."
  },
  "puzzle": {
    "id": "01hgk",
    "rating": 1500,
    "plays": 12345,
    "initialPly": 42,
    "solution": ["e5f7", "g8h8", "f7h6"],
    "themes": ["middlegame", "advantage", "short"]
  }
}
```
</ResponseExample>

<ResponseField name="puzzle.id" type="string">
  5-character puzzle ID
</ResponseField>

<ResponseField name="puzzle.rating" type="integer">
  Puzzle difficulty rating
</ResponseField>

<ResponseField name="puzzle.plays" type="integer">
  Number of times puzzle has been attempted
</ResponseField>

<ResponseField name="puzzle.solution" type="array">
  Array of moves in UCI format
</ResponseField>

<ResponseField name="puzzle.themes" type="array">
  Puzzle themes/tags
</ResponseField>

## Get Puzzle by ID

Get a specific puzzle.

```bash
GET /api/puzzle/{id}
```

<ParamField path="id" type="string" required>
  5-character puzzle ID
</ParamField>

<CodeGroup>
```bash cURL
curl https://lichess.org/api/puzzle/01hgk
```
</CodeGroup>

Returns same format as daily puzzle.

## Get Next Puzzle

Get a new puzzle for the authenticated user.

```bash
GET /api/puzzle/next
```

Requires authentication. Returns a puzzle tailored to the user's rating and preferences.

<ResponseExample>
```json
{
  "game": { },
  "puzzle": {
    "id": "02abc",
    "rating": 1650,
    "plays": 5432,
    "initialPly": 35,
    "solution": ["d4e5", "f6e5", "e1e5"],
    "themes": ["endgame", "queenEndgame"]
  }
}
```
</ResponseExample>

## Get Puzzle Activity

Get recent puzzle activity for the authenticated user.

```bash
GET /api/puzzle/activity
```

Requires authentication.

<ParamField query="max" type="integer" default="50">
  Maximum number of entries (max 100)
</ParamField>

<ParamField query="before" type="timestamp">
  Get entries before this timestamp
</ParamField>

<ResponseExample>

      "id": "01abc",

      "solution": ["f3e5", "c6e5", "d1g4"],

</ResponseExample>

## Batch Select Puzzles

Get a batch of puzzles for a specific angle/theme.

```bash
GET /api/puzzle/batch/{angle}
```

<ParamField path="angle" type="string" required>
  Puzzle theme/angle (e.g., `mix`, `opening`, `middlegame`, `endgame`, or specific theme)
</ParamField>

Returns a JSON object with puzzle IDs for the batch.

## Solve Puzzle Batch

Submit solutions for a batch of puzzles.

```bash
POST /api/puzzle/batch/{angle}
```

<ParamField path="angle" type="string" required>
  Puzzle theme/angle
</ParamField>

<ParamField body="solutions" type="object" required>
  Map of puzzle IDs to results (win/loss)
</ParamField>

Requires `puzzle:write` OAuth scope.

## Puzzle Themes

Common puzzle themes include:

- **Tactical motifs**: `fork`, `pin`, `skewer`, `deflection`, `attraction`, `sacrifice`, `discoveredAttack`, `doubleCheck`, `exposedKing`, `interference`, `intermezzo`, `quietMove`, `xRayAttack`, `zugzwang`

- **Material**: `mate`, `mateIn1`, `mateIn2`, `mateIn3`, `mateIn4`, `mateIn5`, `advantage`, `crushing`, `equality`

- **Game phase**: `opening`, `middlegame`, `endgame`

- **Length**: `oneMove`, `short`, `long`, `veryLong`

- **Piece specific**: `bishopEndgame`, `knightEndgame`, `pawnEndgame`, `queenEndgame`, `queenRookEndgame`, `rookEndgame`

- **Special**: `advancedPawn`, `capturingDefender`, `master`, `masterVsMaster`, `superGM`, `defensive`, `attacking`

## Vote on Puzzle

Vote on puzzle quality.

```bash
POST /training/{id}/vote
```

<ParamField path="id" type="string" required>
  5-character puzzle ID
</ParamField>

<ParamField body="vote" type="integer" required>
  Vote value: 1 (thumbs up) or -1 (thumbs down)
</ParamField>

Requires authentication.

## Report Puzzle

Report a puzzle as incorrect or inappropriate.

```bash
POST /training/{id}/report
```

<ParamField path="id" type="string" required>
  5-character puzzle ID
</ParamField>

<ParamField body="reason" type="string" required>
  Reason for report
</ParamField>

Requires authentication.

## Vote on Puzzle Theme

Vote on whether a theme applies to a puzzle.

```bash
POST /training/{id}/vote/{theme}
```

<ParamField path="id" type="string" required>
  5-character puzzle ID
</ParamField>

<ParamField path="theme" type="string" required>
  Theme name
</ParamField>

<ParamField body="vote" type="boolean" required>
  true if theme applies, false if it doesn't
</ParamField>

Requires authentication.

## Set Puzzle Difficulty

Set preferred difficulty for a puzzle theme.

```bash
POST /training/difficulty/{theme}
```

<ParamField path="theme" type="string" required>
  Theme name
</ParamField>

<ParamField body="difficulty" type="string" required>
  Difficulty: `easiest`, `easier`, `normal`, `harder`, `hardest`
</ParamField>

Requires authentication.

## Complete Puzzle

Submit puzzle solution.

```bash
POST /training/complete/{theme}/{id}
```

<ParamField path="theme" type="string" required>
  Puzzle theme
</ParamField>

<ParamField path="id" type="string" required>
  5-character puzzle ID
</ParamField>

<ParamField body="win" type="boolean" required>
  Whether puzzle was solved correctly
</ParamField>

<ParamField body="rated" type="boolean" default="true">
  Whether attempt should affect rating
</ParamField>

Requires authentication.

## Response Formats

All puzzle endpoints return JSON. Puzzle objects include:

<ResponseField name="id" type="string">
  5-character puzzle ID
</ResponseField>

<ResponseField name="rating" type="integer">
  Puzzle difficulty rating (typically 600-3000)
</ResponseField>

<ResponseField name="plays" type="integer">
  Number of attempts by all users
</ResponseField>

<ResponseField name="initialPly" type="integer">
  Starting ply in the game
</ResponseField>

<ResponseField name="solution" type="array">
  Correct moves in UCI notation
</ResponseField>

<ResponseField name="themes" type="array">
  Array of theme strings
</ResponseField>

<ResponseField name="game" type="object">
  Source game information
</ResponseField>
