## Get User Profile

Get public information about a user.

```bash
GET /api/user/{username}
```

<ParamField path="username" type="string" required>
  Username of the user
</ParamField>

<ParamField query="trophies" type="boolean" default="false">
  Include trophy data
</ParamField>

<ParamField query="challenge" type="boolean" default="false">
  Include whether the user can be challenged
</ParamField>

<ParamField query="profile" type="boolean" default="true">
  Include profile information
</ParamField>

<ParamField query="rank" type="boolean" default="false">
  Include ranking information
</ParamField>

<ParamField query="fideId" type="boolean" default="false">
  Include FIDE player ID if available
</ParamField>

<CodeGroup>
```bash cURL
curl "https://lichess.org/api/user/username?trophies=true&rank=true"
```

```bash Authenticated
curl https://lichess.org/api/user/username \
  -H "Authorization: Bearer {token}"
```
</CodeGroup>

<ResponseExample>

## Get Multiple Users by ID

Get multiple user profiles in one request.

```bash
POST /api/users
```

<ParamField body="usernames" type="string" required>
  Comma-separated list of usernames (max 300)
</ParamField>

<ParamField query="profile" type="boolean" default="true">
  Include profile information
</ParamField>

<ParamField query="rank" type="boolean" default="false">
  Include ranking information
</ParamField>

<CodeGroup>
```bash cURL
curl -X POST https://lichess.org/api/users \
  -H "Content-Type: text/plain" \
  -d "user1,user2,user3"
```
</CodeGroup>

Returns an array of user objects.

## Get Users Status

Get real-time status of multiple users.

```bash
GET /api/users/status
```

<ParamField query="ids" type="string" required>
  Comma-separated user IDs (max 100)
</ParamField>

<ParamField query="withGameIds" type="boolean" default="false">
  Include current game IDs
</ParamField>

<ParamField query="withGameMetas" type="boolean" default="false">
  Include current game metadata
</ParamField>

<ParamField query="withSignal" type="boolean" default="false">
  Include connection lag rating
</ParamField>

<CodeGroup>
```bash cURL
curl "https://lichess.org/api/users/status?ids=user1,user2&withGameIds=true"
```
</CodeGroup>

<ResponseExample>
```json
[
  {
    "id": "user1",
    "name": "User1",
    "online": true,
    "playing": true,
    "streaming": false,
    "playingId": "abcd1234"
  },
  {
    "id": "user2",
    "name": "User2",
    "online": false,
    "playing": false
  }
]
```
</ResponseExample>

## Get Top Players

Get the top players for a performance type.

```bash
GET /api/player/top/{nb}/{perfKey}
```

<ParamField path="nb" type="integer" required>
  Number of players to fetch (max 200)
</ParamField>

<ParamField path="perfKey" type="string" required>
  Performance type: `bullet`, `blitz`, `rapid`, `classical`, `ultraBullet`, `chess960`, `crazyhouse`, `antichess`, `atomic`, `horde`, `kingOfTheHill`, `racingKings`, `threeCheck`
</ParamField>

<CodeGroup>
```bash cURL
curl https://lichess.org/api/player/top/10/blitz
```
</CodeGroup>

<ResponseExample>
```json
{
  "users": [
    {
      "id": "topplayer",
      "username": "TopPlayer",
      "perfs": {
        "blitz": {
          "rating": 3000,
          "progress": 5
        }
      },
      "title": "GM",
      "online": true,
      "patron": true
    }
  ]
}
```
</ResponseExample>

## User Autocomplete

Search for users by username prefix.

```bash
GET /api/player/autocomplete
```

<ParamField query="term" type="string" required>
  Username prefix to search (min 3 characters)
</ParamField>

<ParamField query="object" type="boolean" default="false">
  Return full user objects instead of just usernames
</ParamField>

<ParamField query="friend" type="boolean" default="false">
  Only search followed users
</ParamField>

<CodeGroup>
```bash cURL
curl "https://lichess.org/api/player/autocomplete?term=user&object=true"
```
</CodeGroup>

## Follow/Unfollow User

Follow or unfollow a user.

<CodeGroup>
```bash Follow User
curl -X POST https://lichess.org/api/rel/follow/{username} \
  -H "Authorization: Bearer {token}"
```

```bash Unfollow User
curl -X POST https://lichess.org/api/rel/unfollow/{username} \
  -H "Authorization: Bearer {token}"
```
</CodeGroup>

## Response Fields

<ResponseField name="id" type="string">
  User ID (lowercase username)
</ResponseField>

<ResponseField name="username" type="string">
  Display username
</ResponseField>

<ResponseField name="title" type="string">
  Title (GM, IM, FM, WGM, WIM, WFM, CM, WCM, NM, LM, BOT)
</ResponseField>

<ResponseField name="patron" type="boolean">
  Whether user is a Lichess patron
</ResponseField>

<ResponseField name="verified" type="boolean">
  Whether user is verified
</ResponseField>

<ResponseField name="online" type="boolean">
  Whether user is currently online
</ResponseField>

<ResponseField name="playing" type="string">
  URL of current game if playing
</ResponseField>

<ResponseField name="streaming" type="boolean">
  Whether user is currently streaming
</ResponseField>

<ResponseField name="perfs" type="object">
  Performance ratings for different game types
</ResponseField>

<ResponseField name="count" type="object">
  Game counts (all, rated, win, loss, draw, etc.)
</ResponseField>

<ResponseField name="profile" type="object">
  User profile information (bio, country, location, links, etc.)
</ResponseField>
