## Get Started in 3 Steps

Get your local Lichess development environment up and running quickly.

<Steps>
  <Step title="Clone and Install">
    Clone the repository and ensure you have the prerequisites:
    
    ```bash
    git clone https://github.com/lichess-org/lila.git
    cd lila
    ```
    
    **Prerequisites:**
    - Java JDK 21 or higher
    - MongoDB 4.0+
    - Redis
    - Git
    
    <Accordion title="Installing Prerequisites">
      **On macOS:**
      ```bash
      brew install openjdk@21 mongodb-community redis sbt
      brew services start mongodb-community
      brew services start redis
      ```
      
      **On Ubuntu/Debian:**
      ```bash
      sudo apt install openjdk-21-jdk mongodb redis-server
      sudo systemctl start mongodb
      sudo systemctl start redis
      ```
    </Accordion>
  </Step>

  <Step title="Start the Application">
    Run the Lichess wrapper script which will start sbt:
    
    ```bash
    ./lila.sh
    ```
    
    This creates default configuration files on first run. Inside the sbt console, type:
    
    ```scala
    run
    ```
    
    The application will compile (first time takes 5-10 minutes) and start on `http://localhost:9663`.
    
    <Note>
      The default `.sbtopts` file allocates 2-8GB of RAM. Adjust if needed for your system.
    </Note>
  </Step>

  <Step title="Access Lichess Locally">
    Open your browser and navigate to:
    
    ```
    http://localhost:9663
    ```
    
    You should see the Lichess homepage running locally!
    
    <Tip>
      Create an account on your local instance to test features. The database is isolated from the production site.
    </Tip>
  </Step>
</Steps>

## What Happens on First Run?

When you run `./lila.sh` for the first time:

1. **Configuration files are created** from defaults:
   - `.sbtopts` (SBT memory settings)
   - `conf/application.conf` (app configuration)

2. **Dependencies are downloaded** - all Scala libraries and npm packages

3. **Code is compiled** - Scala 3 compilation of 85+ modules

4. **UI assets are built** - TypeScript compilation and CSS generation

5. **Server starts** on port 9663

## Common sbt Commands

Inside the sbt console:

```scala
run              // Start the application
test             // Run all tests
~compile         // Continuous compilation
project puzzle   // Switch to puzzle module
clean            // Clean build artifacts
reload           // Reload build configuration
```

## Troubleshooting

<Accordion title="Port 9663 already in use">
  Change the port in `conf/application.conf`:
  ```hocon
  http.port = 9664
  ```
</Accordion>

<Accordion title="Out of memory errors">
  Increase heap size in `.sbtopts`:
  ```
  -J-Xmx16g
  ```
</Accordion>

<Accordion title="MongoDB connection failed">
  Ensure MongoDB is running:
  ```bash
  # macOS
  brew services start mongodb-community
  
  # Linux
  sudo systemctl start mongodb
  ```
</Accordion>

<Accordion title="Compilation is slow">
  First compilation takes 5-10 minutes. Subsequent compilations are incremental and much faster.
  
  Consider using Metals (Scala LSP) for IDE support.
</Accordion>

## Playing Your First Local Game

Once the server is running:

1. Visit `http://localhost:9663`
2. Click **Sign Up** to create a local account
3. Click **Play with the computer** or **Create a game**
4. Start playing!

<Note>
  Your local database is completely separate from lichess.org. All accounts and games are isolated.
</Note>
