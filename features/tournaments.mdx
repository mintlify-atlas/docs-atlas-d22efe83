## Overview

Lichess hosts thousands of tournaments daily, from small casual events to large-scale arena competitions with thousands of players. The platform supports multiple tournament formats and includes simultaneous exhibitions (simuls) where one player faces many opponents.

## Tournament Types

### Arena Tournaments

The most popular format on Lichess, Arena tournaments feature:

- **Continuous pairing**: Players are paired as soon as they finish a game
- **Streak bonuses**: Consecutive wins award bonus points
- **Berserk option**: Sacrifice half your time for bonus points
- **Running clock**: Tournament runs for a fixed duration

### Swiss Tournaments

- **Fixed rounds**: Predetermined number of rounds
- **Swiss pairing**: Players with similar scores paired together
- **No elimination**: Everyone plays all rounds
- **Fair progression**: More predictable tournament structure

### Scheduled Tournaments

Lichess runs official tournaments on a fixed schedule:

- **Hourly**: Bullet, Blitz, Rapid
- **Daily**: Various time controls
- **Weekly**: Longer events with special conditions
- **Monthly**: Shield tournaments for top players
- **Yearly**: Annual championships

## Tournament Lifecycle

<Steps>

<Step title="Created">
  Tournament is created and open for registration
  ```scala
  def isCreated = status == Status.created
  def isEnterable = !isFinished
  ```
</Step>

<Step title="Countdown">
  Players join, system displays countdown to start
  ```scala
  def secondsToStart = Seconds((startsAt.toSeconds - nowSeconds).toInt.atLeast(0))
  def imminentStart = isCreated && (startsAt.toMillis - nowMillis) < 1000
  ```
</Step>

<Step title="Started">
  Tournament begins, first round pairings generated
  ```scala
  def isStarted = status == Status.started
  def progressPercent: Int =
    if isCreated then 0
    else if isFinished then 100
    else
      val total = minutes * 60
      val remaining = secondsToFinish.value
      100 - (remaining * 100 / total)
  ```
</Step>

<Step title="Running">
  Players complete games, automatic re-pairing occurs
  ```scala
  def pairingsClosed = secondsToFinish < 
    Seconds(math.max(30, math.min(clock.limitSeconds.value / 2, 120)))
  ```
</Step>

<Step title="Finished">
  Time expires, final standings calculated, winner declared
  ```scala
  def isFinished = status == Status.finished
  def finishedSinceSeconds: Option[Long] = 
    isFinished.option(nowSeconds - finishesAt.toSeconds)
  ```
</Step>

</Steps>

## Tournament Features

### Berserk

Players can "go berserk" at the start of a game:

```scala
// modules/tournament/src/main/Tournament.scala:123
def berserkable = !noBerserk && clock.berserkable

// From game module - clock halved
def goBerserk(color: Color): Option[Progress] =
  clock.ifTrue(game.berserkable && !player(color).berserk)
    .map: c =>
      val newClock = c.goBerserk(color)
      // Player gets half their time but bonus points
```

<Warning>
  Berserk gives 1 bonus point if you win, but you play with half your clock time. Only available for games of 2+ minutes initial time.
</Warning>

### Simul Workflow

<Steps>

<Step title="Host Creates Simul">
  ```scala
  def make(
      host: UserWithPerfs,
      name: String,
      clock: SimulClock,
      variants: List[Variant],
      color: String  // Which color host plays
  ): Simul
  ```
</Step>

<Step title="Players Apply">
  ```scala
  def addApplicant(applicant: SimulApplicant) =
    Created:
      if !hasApplicant(applicant.player.user) && 
         variants.has(applicant.player.variant)
      then copy(applicants = applicants :+ applicant)
  ```
</Step>

<Step title="Host Accepts Players">
  ```scala
  def accept(userId: UserId, v: Boolean) =
    Created:
      copy(applicants = applicants.map: a =>
        if a.is(userId) then a.copy(accepted = v) else a)
  
  def nbAccepted = applicants.count(_.accepted)
  def startable = isCreated && nbAccepted > 1
  ```
</Step>

<Step title="Simul Starts">
  ```scala
  def start =
    startable.option(
      copy(
        status = SimulStatus.Started,
        startedAt = nowInstant.some,
        applicants = Nil,
        clock = clock.adjustedForPlayers(nbAccepted),
        pairings = applicants.collect:
          case a if a.accepted => SimulPairing(a.player)
      )
    )
  ```
</Step>

<Step title="Games Played">
  Host plays all games simultaneously, moving between boards
  ```scala
  def updatePairing(gameId: GameId, f: SimulPairing => SimulPairing) =
    copy(pairings = pairings.map(p => if p.gameId == gameId then f(p) else p))
      .finishIfDone
  ```
</Step>

<Step title="Simul Completes">
  ```scala
  private def finishIfDone =
    if isStarted && pairings.forall(_.finished) then
      copy(
        status = SimulStatus.Finished,
        finishedAt = nowInstant.some,
        hostGameId = none
      )
  
  def wins = pairings.count(p => p.finished && p.wins.has(false))
  def draws = pairings.count(p => p.finished && p.wins.isEmpty)
  def losses = pairings.count(p => p.finished && p.wins.has(true))
  ```
</Step>

</Steps>

### Simul Scoring

```scala
// modules/simul/src/main/Simul.scala:140
def wins = pairings.count(p => p.finished && p.wins.has(false))   // Host wins
def draws = pairings.count(p => p.finished && p.wins.isEmpty)
def losses = pairings.count(p => p.finished && p.wins.has(true))  // Host losses  
def ongoing = pairings.count(_.ongoing)

def hostScore = lila.core.rating.Score(wins, losses, draws, none)
def playerScore(userId: UserId) = pairingOf(userId).map: p =>
  Score(
    p.wins.has(true).so(1),   // Beat the host = 1
    p.wins.has(false).so(1),  // Lost to host = 0 
    p.wins.isEmpty.so(1),     // Drew host = 0.5
    none
  )
```
