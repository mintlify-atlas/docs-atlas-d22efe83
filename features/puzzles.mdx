## Overview

Lichess offers over 3 million chess puzzles extracted from real games, all rated using a Glicko-2 rating system. The puzzles adapt to your skill level and provide comprehensive tactical training across various themes.

## Puzzle Data Model

```scala
// modules/puzzle/src/main/Puzzle.scala:7
case class Puzzle(
    id: PuzzleId,
    gameId: GameId,              // Source game
    fen: Fen.Full,               // Starting position
    line: NonEmptyList[Uci.Move], // Solution sequence
    glicko: Glicko,              // Puzzle rating
    plays: Int,                   // Times attempted
    vote: Float,                  // Quality score (0-1)
    themes: Set[PuzzleTheme.Key] // Tactical themes
):
  def initialPly: Ply = Fen.readPly(fen) | Ply.initial
  
  def color = !fen.colorOrWhite  // Side to move
  
  lazy val fenAfterInitialMove: Fen.Full =
    boardAfterInitialMove.map(Fen.write)
      .err(s"Can't apply puzzle $id first move")
```

### Puzzle Components

- **FEN**: Position after opponent's move (your turn)
- **Solution line**: Sequence of moves (you, opponent, you, etc.)
- **Rating**: Glicko-2 rating indicating difficulty
- **Themes**: Tactical patterns (fork, pin, sacrifice, etc.)
- **Vote**: Community quality rating

## Puzzle Generation

Puzzles are automatically extracted from Lichess games:

1. **Game Analysis**: Games analyzed by Stockfish
2. **Pattern Detection**: Critical moments identified (tactics, mistakes)
3. **Validation**: Solution verified for accuracy
4. **Theme Assignment**: Tactical patterns tagged
5. **Quality Control**: Community voting on puzzle quality

<Info>
  Puzzles come from games at all rating levels, providing training material for players from beginner to grandmaster.
</Info>

## User Progress Tracking

The system tracks each user's puzzle performance:

```scala
// modules/puzzle/src/main/PuzzleApi.scala:44
private[puzzle] object round:
  
  def find(user: User, puzzleId: PuzzleId): Fu[Option[PuzzleRound]] =
    colls.round(_.byId[PuzzleRound](
      PuzzleRound.Id(user.id, puzzleId).toString
    ))
  
  def upsert(r: PuzzleRound, angle: PuzzleAngle): Funit =
    val roundDoc = roundHandler.write(r) ++
      $doc(
        PuzzleRound.BSONFields.user -> r.id.userId,
        PuzzleRound.BSONFields.angle -> angle.some.filter(_ != PuzzleAngle.mix)
      )
    colls.round(_.update.one($id(r.id), roundDoc, upsert = true)).void
```

### Round Data

Each puzzle attempt stores:
- **Win/Loss**: Whether the puzzle was solved
- **Rating change**: Glicko-2 update for both user and puzzle
- **Timestamp**: When attempted
- **Theme votes**: User feedback on puzzle themes

## Rating System

Both puzzles and users have Glicko-2 ratings:

```scala
case class Puzzle(
  glicko: Glicko,  // Contains rating, deviation, volatility
  // ...
)

case class UserResult(
    puzzleId: PuzzleId,
    userId: UserId,
    win: PuzzleWin,
    rating: (IntRating, IntRating)  // (before, after)
)
```

### Rating Updates

- **Win**: Both player and puzzle ratings adjust based on expected outcome
- **Loss**: Ratings adjust inversely
- **Provisional**: New puzzles/players have higher rating volatility
- **Convergence**: Ratings stabilize with more attempts

<Tip>
  The Glicko-2 system ensures puzzles are matched to your skill level, providing an optimal learning challenge.
</Tip>

### Theme Voting

```scala
// modules/puzzle/src/main/PuzzleApi.scala:136
def theme.vote(id: PuzzleId, themeStr: String, vote: Option[Boolean])(
    using me: Me
): FuRaise[PuzzleTheme.VoteError, Unit] =
  PuzzleTheme.findDynamic(themeStr)
    .raiseIfNone(Fail(s"Unknown theme $themeStr"))
    .flatMap: theme =>
      for
        puzRound <- round.find(me, id)
        puzRound <- puzRound.raiseIfNone(Fail(s"Puzzle $id not yet played"))
        newThemes <- PuzzleRound.themeVote(puzRound.themes)(theme.key, vote)
                      .raiseIfNone(Unchanged)
        _ <- update.isDefined.so:
          colls.puzzle(_.updateField($id(puzRound.id.puzzleId), 
                                     Puzzle.BSONFields.dirty, true))
      yield lila.mon.puzzle.vote.theme(theme.key.value, vote, puzRound.win.yes)
              .increment()
```

Users can vote on whether a theme applies to a puzzle:
- **Upvote**: Theme is accurate
- **Downvote**: Theme doesn't apply
- **Remove vote**: Undo previous vote

## Casual Mode

Users can solve puzzles without rating impact:

```scala
// modules/puzzle/src/main/PuzzleApi.scala:177
object casual:
  
  private val store = scalalib.cache.ExpireSetMemo[String](30.minutes)
  
  def setCasualIfNotYetPlayed(user: User, puzzle: Puzzle): Funit =
    round.exists(user, puzzle.id).not
      .mapz(store.put(key(user, puzzle.id)))
  
  def apply(user: User, id: PuzzleId) = store.get(key(user, id))
```
